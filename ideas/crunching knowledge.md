# crunching knowledge

Developers should learn domain from domain experts. They build a model - the abstraction representing the domain. The model should be useful in a sense that developers and experts understand it. So it can be used to represent the knowable and also to test an implementation. The model is not static. It’s changing when all participants discover new details and aspects. Knowledge crunching is an exploration, and you can’t know where you will end up.

  

The model and quick prototyping is used to get quick feedback from domain experts.
