# hiding details

The model should have all important details. Sometimes the implementation has some if statements implementing business rules. In case the rule is not represented in the model the domain expert can’t read it. So the rule is hidden in the implementation. Someone who learn the project or change the rules can’t know about this implementation detail, so it makes interaction of knowledge complicated.
