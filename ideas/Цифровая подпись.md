# Цифровая подпись

# Как работает цифровая подпись  
## На примере JWT и отличие от шифрования

---

## 1. Что такое цифровая подпись

**Цифровая подпись** решает две задачи:

1. **Аутентичность** — кто создал сообщение  
2. **Целостность** — не было ли сообщение изменено  

❗ **Цифровая подпись НЕ шифрует данные** — содержимое остаётся читаемым.

---

## 2. Базовые компоненты

Цифровая подпись всегда использует:

- **Хэш-функцию**
  - SHA-256, SHA-512 и т.д.
  - Создаёт короткий «отпечаток» данных
- **Криптографический ключ**
  - Симметричный (HMAC)
  - Асимметричный (RSA, ECDSA)

---

## 3. Как работает цифровая подпись (шаги)

### Подписание

Есть сообщение:

    M = "user_id=42&role=admin"

1. Считаем хэш:

    H = hash(M)

2. Создаём подпись:

**HMAC**

    signature = HMAC(secret_key, H)

**RSA / ECDSA**

    signature = sign(private_key, H)

3. Отправляем:

    M + signature

| Свойство              | HMAC                | RSA / ECDSA                 |
| --------------------- | ------------------- | --------------------------- |
| Тип ключа             | Симметричный        | Асимметричный               |
| Кто может подписывать | Все, кто знает ключ | Только владелец private key |
| Проверка              | Тем же ключом       | Public key                  |
| Подходит для JWT      | ✅ HS256             | ✅ RS256                     |

---

### Проверка подписи

Получатель:

1. Считает хэш сам:

    H' = hash(M)

2. Проверяет подпись:
- HMAC — тем же секретным ключом  
- RSA / ECDSA — публичным ключом  

3. Если:

    H == H'

→ сообщение подлинное и неизменённое

---

## 4. JWT как пример цифровой подписи

смотри так же [[В чем суть JSON Web Token и зачем он]]

JWT состоит из трёх частей:

    header.payload.signature

---

### Пример JWT

**Header**

    {
      "alg": "HS256",
      "typ": "JWT"
    }

**Payload**

    {
      "sub": "42",
      "role": "admin",
      "exp": 1710000000
    }

---

## 5. Чем цифровая подпись отличается от шифрования

| Свойство | Цифровая подпись | Шифрование |
|--------|------------------|------------|
| Скрывает данные | ❌ Нет | ✅ Да |
| Гарантирует автора | ✅ Да | ❌ Нет |
| Защищает от изменения | ✅ Да | ❌ Нет |
| Можно читать без ключа | ✅ Да | ❌ Нет |
| Основная цель | Доверие | Секретность |

---

## 6. Коротко

- JWT **подписывается**, а не шифруется  
- Подпись = доверие и защита от подмены  
- Base64 ≠ шифрование
