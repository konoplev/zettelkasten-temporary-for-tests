# Как устанавливается tcp соединение и что такое SYN SYN-ACK ACK (3-стороннее рукопожатие)

![[files/telegram-cloud-photo-size-2-5406904732771016458-x.jpg]]

По tcp соединению пакеты шлются с порядковыми номерами. В каждом направлении своя последовательность номеров.


SYN - специальное сообщение, в котором узел сообщает другой стороне свой **начальный порядковый номер**.
  

ACK - специальное сообщение, которым узел сообщает другой стороне, что узлом были получены все пакеты вплоть до такого-то номера.

SYN шлется только при установке соединения.

ACK шлются на каждый пакет.

Нужность ACK очевидна.

Нужность SYN не особо. Можно ведь начинать порядковый номер с нуля. Так раньше и было. Сейчас начальный порядковый номер задается случайно. Основная причина - security, так сложнее хакнуть соединение.

  
Итого, клиент и сервер говорят друг другу SYN со своими начальными порядковыми номерами. И отправляют друг другу ACK на эти SYN. **Обмен номерами состоялся - соединение установлено.**

SYN-ACK посередине отправляются совместно для экономии времени.
