# Wide column и key value бд — в чем разница?


Выше я написал, что DynamoDB от Amazon - wide column store, как Big table. Кажется, это ошибка, и DynamoDB - просто key value.

А в чем разница между key value и wide column?  

На вики пишут, что wide column - это когда в разных строках могут быть разные колонки. На самом деле это неполное определение и не до конца объясняет, что происходит в индустрии. 


Многие key value базы позволяют работать с атрибутами объекта, который вы залили в качестве value:

- можно запросить только определенные поля 

- можно добавить определенные поля, не вычитывая перед этим весь value

И атрибуты будут разные для разных key. Чем не wide column?

Так вот, полное определение Wide column.

Wide column - это 

а) когда кол-во колонок **неограниченно.** То есть у вас не одномерная мапа `map[key]->value`, а двумерная `map[key][column]->value` и кол-во column неограниченно - прямо как кол-во key.

б) когда у вас есть возможность управлять хранением колонок: какие колонки хранятся вместе, какие отдельно, какие в память надо грузить, какие нет и тд, и тп.

Всё это невозможно в случае с key value, которые хранят value одним blob-ом.

DynamoDB хоть и позволяет работать с атрибутами у value, позволяет по разным key иметь разные атрибуты у value, но ни а), ни б) не позволяет. А потому DynamoDB - не wide column.
